# Wave 4: E2E 覆盖率提升到 70%

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development

**Goal:** E2E 覆盖率从当前 ~30% 提升到 70%，覆盖所有关键用户旅程。

**Architecture:** 合并 designer-polish worktree 的 E2E；Engine Cypress 替换 mock token 为真实登录；补齐 UI CRUD + 发布流程 E2E。

---

## §1 当前 E2E 缺口

| 用户旅程 | 当前覆盖 | 需补 |
|---------|:---:|------|
| 登录 | mock token only | 真实登录 + 错误密码 + 登出 |
| 站点 CRUD | API only | UI 创建/编辑/删除 |
| 页面 CRUD | read only | UI 创建/编辑/删除 |
| 页面编辑器 | 无（worktree 有） | 合并 designer-drag + designer-edit |
| 发布/下线 | API only | UI 发布按钮 + 状态变化 + 访客可见 |
| 用户管理 | smoke only | UI 创建/编辑/角色/删除 |
| 设置 | smoke only | UI 保存 |
| 公开页面渲染 | ✅ good | 保持 |
| 留资捕获 | ✅ good | 保持 |

---

## Tasks

### T1: 合并 designer-polish E2E

- 从 `.worktrees/designer-polish/packages/engine/luban/cypress/e2e/` 合并：
  - `designer-drag-*.cy.ts`（拖拽测试）
  - `designer-edit.cy.ts`（属性编辑测试）
- 合并到 `packages/engine/luban/cypress/e2e/`

### T2: Engine Cypress 真实登录替换 mock token

- Modify: `cypress/support/commands.ts` — 替换 `loginWithToken('mock-jwt-token')` 为真实 POST /api/auth/login
- Modify: `cypress.config.ts` — 配置 baseUrl 指向真实 engine + BFF
- 需确保 `make e2e-up` 启动的全栈环境可用

### T3: 站点 CRUD UI E2E

- Create: `cypress/e2e/sites-crud.cy.ts`
  - 创建站点（填表 → 提交 → 列表出现）
  - 编辑站点（修改名称 → 保存 → 列表更新）
  - 删除站点（确认 → 列表消失）

### T4: 页面 CRUD UI E2E

- Create: `cypress/e2e/pages-crud.cy.ts`
  - 创建页面（填名称/路径 → 保存 → 编辑器打开）
  - 编辑页面（修改名称 → 保存）
  - 删除页面（确认 → 列表消失）

### T5: 发布/下线 UI E2E

- Create: `e2e/flows/publish-ui.spec.ts`（Playwright，跨项目）
  - 在 PageEditor 点击发布 → status 变 published
  - 访问 Website 公开 URL → 页面可见
  - 在 PageList 点击下线 → status 变 archived
  - 访问 Website 公开 URL → 404
  - 版本历史 → 回滚 → 草稿恢复

### T6: 用户管理 UI E2E

- Create: `cypress/e2e/users-crud.cy.ts`
  - 创建用户（填表 → 提交）
  - 编辑用户角色
  - 修改状态（启用/禁用）
  - 删除用户

### T7: 设置保存 UI E2E

- Create: `cypress/e2e/settings-save.cy.ts`
  - 修改设置 → 保存 → 刷新页面 → 设置持久

### T8: 登录真实流程 E2E

- Create: `cypress/e2e/auth-flow.cy.ts`
  - 正确登录 → 跳转 dashboard
  - 错误密码 → 错误提示
  - 登出 → 跳转 login
  - 刷新页面 → 保持登录（持久化验证）

### T9: 自动化 seed 脚本

- Create: `scripts/e2e/seed.sh` — 自动创建测试账号 + 默认站点/页面
- Modify: `scripts/e2e/up-all.sh` — 启动后自动执行 seed

### T10: CI E2E 门禁化

- Modify: `.github/workflows/e2e-cross.yml` — 移除 `continue-on-error: true`

make e2e-up  # 启动全栈

关键旅程覆盖率 ≥ 70%（9 个旅程中 ≥ 7 个有完整 E2E）。
