但以下三项在 round-1 审查中标记为 🔴/🟡（未在 wave-1 落地，显式延后）：

## 范围

### R2 — PropertyPanel 缺「管理数据源」+「测试连通」按钮（§4.3）
- 计划 §4.3 ASCII 图明列 `[管理数据源] [测试连通]`
- 数据源分区当前只有 ElSelect + varName ElInput，无两个按钮
- `src/api/datasource.ts` 的 testDatasource/createDatasource/updateDatasource/deleteDatasource 已实现但 PropertyPanel 未消费
- **需补**：
  - 「管理数据源」按钮 → 弹层 ElTable 展示 datasource 列表，支持新建/编辑/删除（调 CRUD）
  - 「测试连通」按钮 → 调 testDatasource(id)，显示结果 {ok, message, latencyMs}

### R3 — PropertyPanel 四态缺「错」「成功」态（§4.3）
- 计划 §4.3「四态：加载(v-loading)/空(未选中提示)/错(表达式/连通失败+重试)/成功」
- 当前仅实现空态 + 默认表单，无 v-loading / 错误卡片 / 成功反馈
- PageEditor.vue:384 已有 v-loading 和错误卡片+重试（达标），PropertyPanel 需对齐
- **需补**：
  - 数据源区/事件区加 v-loading 容器
  - 表达式输入或连通测试失败时显示错误卡片 + 重试按钮
  - 操作成功时显示成功反馈

### Y3 — ComponentTree 缺锁定/隐藏态（§4.2#7）
- 计划 §4.2#7「锁定节点(L)→不可拖/删/改；隐藏节点(H)→编辑态不渲染保留 schema」
- ComponentTree.vue 无 locked/hidden 图标标识；locked 节点删除按钮未禁用
- useKeyboard 仅 undo/redo/save/delete/duplicate，无锁定(L)/隐藏(H) 快捷键
- **注**：low-code DesignRenderer 已做 locked 编辑态过滤（round-1 已修 filter），
  本项补 engine ComponentTree 的 UI 展示 + 快捷键

- 锁定(L)/隐藏(H) 快捷键生效
