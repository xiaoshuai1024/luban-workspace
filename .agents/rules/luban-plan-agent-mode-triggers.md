# Plan / Agent 模式切换（用户用语触发）

## MUST：识别用户意图后立即调整工作模式

本仓库对话中，**在回复正文之前**（或在与任务无关的澄清问题之前），根据用户**当前轮**自然语言**根据用户意图调整工作模式（方案讨论 vs 代码执行）**，并给出一句话说明调整原因。

### 切到 **Plan**（方案讨论模式）

当用户表达**要先设计、讨论、拆解、定稿**而非直接改代码时触发，包括但不限于：

- **方案**：写方案、做方案、出方案、定方案、补方案、改方案、技术方案、设计方案、架构方案、落地方案、实施计划、迭代计划
- **规划**：规划、路线图、roadmap、可行性、评审方案、方案设计、做技术设计
- **与「只出方案」同类的表述**：先别写代码、先把思路理清、帮我列步骤、有哪些 trade-off、给一版设计

若用户**明确**说「只讨论 / 不要改仓库 / 先出文档」，一律切 **Plan**。

### 切到 **Agent**（代码执行模式）

当用户表达**要在仓库里动手实现、跑命令、改代码、按已定稿执行**时触发，包括但不限于：

- **执行**：执行、实施、落地、开发、实现、开写、开始写代码、按方案做、按计划做、照着做、跑一下、修一下、改代码、提交、开 PR

若当前轮**以执行收尾**（例如「方案定了，执行吧」），切 **Agent**。

### 冲突与同一条消息多意图

- **先方案后执行**（同一条里顺序明确）：先 **Plan**；用户在下一条再说「执行」时再 **Agent**。
- **同时要求又写新方案又立刻改码**：默认 **Plan**，并在切换说明里提示：是否拆成「先定稿再实现」两阶段。
- **纯问答**（与方案/执行无关）：**不**强制切换。

## DO / DON'T

- **DO**：触发语出现时**主动**调整工作模式，不要等用户手动指示。
- **DO**：调整说明写清是依据哪类关键词（方案类 / 执行类）。
- **DON'T**：在已处于目标模式时重复调整（除非用户再次用触发语强调要换模式）。
