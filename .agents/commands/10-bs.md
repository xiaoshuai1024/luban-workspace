---

触发：**`/10-bs`**。Agent **必须**把头脑风暴的**最终综合结论**写入**指定的方案文件**，禁止仅在聊天中输出而不落盘。

### 1. 定位「当前正在执行的方案」文档（目标文件）

按优先级确定要写入的 **单个** Markdown 路径：

1. 用户在**本条消息或上文**中给出的路径（含 `@文件` 附件）；
2. 当前会话中明确在迭代的那份方案（例如 `.agents/plans/*.plan.md`、`docs/**/*.md` 中带「方案 / 计划 / Plan」且正在修改的文件）；
3. 若仍无法唯一确定：**询问用户**一次，让其粘贴路径或从常用方案中选一项。

无可靠目标文件时 **不得**随便写入任意文档。

### 2. 读取方案并提炼命题

阅读目标文件全文（或相关章节），提炼：**命题边界**、已有决策、待决项、与代码/文档的引用关系。

### 3. 并行执行「十轮」（含义与落盘范围）

**并行十轮** = 十个 **彼此独立** 的审视视角在**内部**同时审视**同一命题**，不是时间上先后改稿十次。

- Agent 在推理中应覆盖十类视角（见下表「维度池」），保证综合结论**不漏维度**。
- **落盘约束**：目标 Markdown **只写综合结论**，**不得**写入每一轮的聚焦、论证表格、分轮小结等过程材料。分轮过程如需沟通，仅在聊天中用极简摘要即可（可选）。

推荐维度**池**（内部审视用；可从池中选满 10 个视角再合并为结论）：

| # | 视角示例 |
|---|----------|
| 1 | 业务闭环 / 用户与运营是否走得通 |
| 2 | 技术可行性与边界条件 |
| 3 | 多端一致（引擎渲染/website/各 client 渲染一致） |
| 4 | 安全、合规、敏感信息 |
| 5 | 性能、容量、成本 |
| 6 | 运维、发布、回滚 |
| 7 | 可观测性、排障、日志/监控 |
| 8 | 与 meta 仓既有子项目/架构对齐 |
| 9 | 契约、API、版本兼容（双后端一致） |
| 10 | 风险、里程碑、未决依赖 |

若命题偏纯工程（如日志方案），可将池替换为：**磁盘 / grep / 降噪 / requestId / 异步 / 容器 / 合规 / 演进路线 / 排障 / 与代码映射** 等独立切口。

### 4. 写入目标文档的结构（必须落盘）

在目标文件中新增或替换 **`## 十轮并行头脑风暴与结论`**（或与仓库内同类文档标题一致），**仅**包含以下块（顺序固定）：

1. **一段说明**（2～4 句即可）：何谓「并行十轮」；明确 **本文仅收录综合结论、不含各轮过程**。
2. **`## 综合结论`**：合并后的条目列表或精简表格；应能覆盖上节维度池中的要点，**不**以「第 N 轮」为小标题展开。
3. 可选：**与本文档前文「待办 / 阶段 / 检查清单」的映射**（简短）。
4. **`## 修订记录`**：日期、`/10-bs` 生成或更新说明。

**禁止写入目标文件**：`### 第 N 轮 · …`、各轮「聚焦 / 论证 / 本轮结论」、冗长辩论过程。

**体例说明**：仓库内历史文档可能含分轮展开，**新一次 `/10-bs` 触发的落盘**一律按本节「仅综合结论」执行；若替换旧章节，删除分轮过程仅保留符合本节结构的精炼内容。

### 5. 与已有章节的关系

- 若文档中 **尚无** 该章节：插在全文 **文末**或「依赖 / 执行顺序」一节 **之前**（由 Agent 根据文档结构选择最不破坏阅读的一处）。
- 若 **已有**「十轮头脑风暴」类章节：**整体替换**为本轮产出（综合结论体例），并在修订记录中注明「由 `/10-bs` 更新」。

### 6. Agent 自检（结束前）

- [ ] 已写入**磁盘上的目标 Markdown**，路径与用户预期一致
- [ ] 目标文件中 **无** 分轮过程章节，仅有「说明 + 综合结论（+ 可选映射）+ 修订记录」
- [ ] 综合结论可被方案的「下一阶段 / 待办」直接引用，且维度覆盖完整

策略：本命令与 GitHub/PR 无关；不运行破坏性脚本。
