# 假功能（Fake Feature）定义与禁止清单

> 统一定义「假功能」的概念、分类、识别方法和禁止规则，作为产品质量审查与 E2E 验收的统一依据。
> 通用，适用 luban 全栈（引擎/BFF/双后端/多端）。

---

## 1. 定义

**假功能**：在用户可见的界面中，存在交互入口或功能承诺，但实际执行时缺乏完整业务逻辑、数据承接或状态回流，导致用户无法完成预期目标的实现方式。

核心判定标准：**用户点击/操作后，是否产生了可追踪的业务状态变化，且该变化可被后续流程消费。**

---

## 2. 分类与示例

### 2.1 弹窗代替逻辑（Alert-Only）

| 表现 | 说明 | 正确做法 |
|------|------|---------|
| 点击按钮仅弹出提示「操作成功」，无后续状态变化 | 无数据写入、无接口调用、无页面跳转 | 调用后端接口 → 状态回流 → UI 更新 |
| 删除操作仅弹 confirm 后提示「已删除」，但数据未真删 | 仅前端视觉变化，刷新即恢复 | 调用删除接口 → 列表重新拉取 → 确认消失 |
| 保存操作仅提示「保存成功」，但表单数据未提交 | 前端假保存 | 提交到后端 → 返回确认 → 列表/详情刷新 |

**一票否决条款**：点击只有弹窗提示无法继续流程 / 关键状态无回流 / 关键按钮无可追踪结果 → 禁止标记「已完成」。

### 2.2 占位入口（Placeholder Entry）

| 表现 | 说明 | 正确做法 |
|------|------|---------|
| 按钮可点击但 `onclick=""` 或空函数 | 无任何处理逻辑 | 绑定真实事件处理函数 |
| 菜单项/Tab 存在但点击后无内容或显示「Coming Soon」 | 伪入口 | 实现完整功能或移除入口 |
| 链接 `href="#"` 静默无响应 | 死链 | 若暂不可用，点击后说明「未在当前版本范围」 |
| 卡片/入口仅静态展示，无可操作结果 | 仅 UI 壳 | 实现操作逻辑或移除 |

### 2.3 假数据冒充（Fake Data）

| 表现 | 说明 | 正确做法 |
|------|------|---------|
| 列表页使用硬编码假数据行冒充真实数据 | 前端写死数据 | 接入真实 API，空态用空态说明 + 引导 |
| 详情页展示固定 mock 数据 | 不随 ID 变化 | 从后端拉取对应记录 |
| 表单提交到 mock 接口或直接忽略 | 数据不入库 | 提交到真实后端接口 |

### 2.4 骨架页冒充完成（Skeleton-as-Done）

| 表现 | 说明 | 正确做法 |
|------|------|---------|
| 页面仅渲染骨架屏，无真实内容 | 冒充已实现 | 实现真实数据加载与展示 |
| 列表无限 skeleton 加载冒充有数据 | 无数据时无空态 | 空态说明 + 引导操作 |

### 2.5 演示版本（Demo-Only）

| 表现 | 说明 | 正确做法 |
|------|------|---------|
| 以「MVP」「先占位」「仅 E2E 能跑」「门禁绿即可」为口径 | 仅够联调/演示 | 全量实现，满足交付后即可使用 |
| 仅主流程可走通，省略方案所列全部用户可见能力 | 缩水交付 | 一次完成方案全部范围 |
| 仅后端接口可用或仅集成测通过 | 无前端闭环 | 全栈交付 |
| 将同一已定稿方案拆成多期交付 | 分期缩水 | 一次完成该 plan 与任务图所载范围 |

### 2.6 假价/不可达入口（Fake Price / Unreachable Entry）

| 表现 | 说明 | 正确做法 |
|------|------|---------|
| 营销页面展示不可达的优惠入口 | 用户无法享受 | 确保入口可达且数据一致 |
| 价格/优惠与实际不一致 | 假价 | UI 与数据一致 |
| 模块关闭后仍展示入口 | 死入口 | 模块关闭时隐藏或禁用入口 |

### 2.7 硬编码配置（Hardcoded Config）

| 表现 | 说明 | 正确做法 |
|------|------|---------|
| 业务参数硬编码在代码中 | 不可配置 | 配置化/模板化 |
| 色彩值硬编码 | 不走设计系统 | 使用设计 token/变量 |

### 2.8 假绿（Fake Green）

| 表现 | 说明 | 正确做法 |
|------|------|---------|
| E2E 测试全体 skip + 退出码 0 冒充通过 | 未真实验证 | 真实断言，首个失败即停 |
| 弱化断言（仅检查 200 状态码） | 不验证业务逻辑 | 断言业务数据与状态 |
| 滥用 `SKIP_*` 环境变量 | 跳过门禁 | 仅 CI 豁免场景使用 |
| 改默认账号敷衍过关 | 不验证真实权限 | 使用正确测试账号 |

### 2.9 引擎/物料特有假功能（luban）

| 表现 | 说明 | 正确做法 |
|------|------|---------|
| 物料 props schema 不完整，渲染时用默认值冒充 | 引擎渲染与设计不符 | schema 完整定义所有 props |
| 引擎渲染某端正常，另一端降级/空白 | 多端不一致 | 各端渲染一致 |
| BFF 聚合字段缺失，用前端 fallback 填充 | BFF 假聚合 | BFF 完整聚合 |

## 3. 审查时的判定流程

```
发现可疑功能入口
    │
    ├─ 点击/操作后是否有可追踪的业务状态变化？
    │   ├─ 否 → 假功能（阻断）
    │   └─ 是 → 继续
    │
    ├─ 状态变化是否可被后续流程消费？
    │   ├─ 否 → 假功能（阻断）
    │   └─ 是 → 继续
    │
    ├─ 数据来源是否为真实后端接口？
    │   ├─ 否（硬编码/mock）→ 假功能（阻断）
    │   └─ 是 → 继续
    │
    ├─ 双后端（Java/Go）行为是否一致？
    │   ├─ 否 → 假功能（阻断，契约不一致）
    │   └─ 是 → 继续
    │
    └─ 是否覆盖方案/PRD 所列全部能力？
        ├─ 否 → 缩水交付（阻断）
        └─ 是 → 通过
```

---

## 4. 验收清单

每次交付前自检：

- [ ] 所有可点击入口都有真实业务逻辑？
- [ ] 所有写操作都有状态回流（API → DB → 可查询）？
- [ ] 所有展示数据来自真实后端（无硬编码假数据）？
- [ ] 所有列表/详情都有空态处理（非无限 skeleton）？
- [ ] 覆盖方案所列全部能力（无缩水）？
- [ ] 双后端（Java/Go）行为一致？
- [ ] 多端（electron/flutter/web）业务一致？
- [ ] 引擎渲染零新增 console error？
- [ ] E2E 真实断言（无假绿）？
