# prototype/

快速原型（Spike）提示词模板。用于：在投入大规模实现前，**用最小代价验证一个技术假设是否成立**。

## 何时用

- 想接入一个新的 luban-ui 物料，但不确定 schema / 渲染管线能否承载 → 先写一个最小物料 + 一个渲染 demo 页面。
- 想给 BFF 加一个聚合接口，但不确定 Java/Go 双后端的响应体差异 → 先用 curl / mock 各打一遍对齐契约。
- 想验证引擎产物在 luban-website（SSR）与各端渲染是否一致 → 先做一个「同一 schema 多端跑」的对照 demo。

## 产出标准

一个 prototype 在结束时**必须**给出：

1. **结论**：假设是否成立？YES / NO / 部分成立（带条件）。
2. **证据**：可运行的代码片段、console 输出、curl 响应、截图、coverage 报告片段之一即可——必须有物证，禁止纯口述。
3. **下一步建议**：若成立 → 转入 `plan/` 走正式实现；若不成立 → 列出阻塞点与替代方案；若部分成立 → 明确补充条件。

## 反模式（禁止）

- ❌ 「我先搭个脚手架看看」然后无结论地铺开 → 不是 prototype，是失控的实现。
- ❌ 用 mock 数据冒充真实渲染 / 真实接口结果。
- ❌ prototype 没结束就擅自合并进默认分支。
