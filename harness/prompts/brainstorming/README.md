# brainstorming/

头脑风暴 / 讨论稿提示词模板。用于：**正式方案之前的发散阶段**——列想法、假设、待确认问题、明确不做，先对齐再收敛。

## 与 plan/ 的区别

| 维度 | brainstorming/ | plan/ |
|------|----------------|-------|
| 目的 | 发散、对齐范围 | 收敛、可落盘定稿 |
| 产出 | 讨论稿（不落盘） | `docs/superpowers/plans/<feature>.md` |
| 何时停 | 用户拍板范围 | 写完定稿 + 通过 plan 门禁 skill 核对 |
| 严格度 | 自由 | 须满足方案契约 |

## 何时用

- 需求模糊，需要先澄清「到底要做什么」。
- 一个想法可能涉及多个方向，想先列全再砍。
- 用户说「我们讨论一下 X 可行性」「帮我想想怎么拆」。

## 何时**不**用

- 需求已清晰 → 直接走 `plan/`。
- 1–2 文件小改动 → 直接执行。

## 反模式

- ❌ 头脑风暴里就开始写代码。
- ❌ 把发散当定稿，直接落盘 `plans/*.md`。
- ❌ 列了一堆假设但没编号，用户无法逐条回复。
