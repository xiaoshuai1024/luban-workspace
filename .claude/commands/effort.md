---
description: 设定本会话推理/执行用力档位（等价 Claude Code /effort）
---

触发：**`/effort`**，可选参数：`low` | `medium` | `high` | `xhigh` | `max` | `auto`。

在 Cursor 中通常**无法**像 Claude Code CLI 那样改写底层 `model_reasoning_effort`；本命令约定 **本会话剩余轮次的行为档位**，并与用户所选档位对齐。若未带参数，Agent 应先 **AskQuestion** 或列出下列档位请用户选一项（单选），再宣告生效。

## 档位定义（行为契约）

| 档位 | 推理与表达 | 工具与验证 | 默认倾向 |
| --- | --- | --- | --- |
| **low** | 最短路径回答；少展开背景 | 仅用户明确要求或明显风险时才跑命令/多读文件 | 快、省 token |
| **medium** | 平衡说明与结论；关键处简短推理 | 需要时读相关文件；验证以用户明确要求为准 | 日常开发默认 |
| **high** | 多步自检；边界与替代方案点到为止 | 更主动 grep/read；关键路径倾向跑相关检查 | 合并前 / 复杂缺陷 |
| **xhigh** | 系统性排查；显式列出假设与反证 | 更完整上下文收集；更倾向跑测试/类型/构建（若仓库惯例） | 大改动、安全/资金路径 |
| **max** | 在 xhigh 基础上仍不放过低概率风险；篇幅可更长 | 验证更激进（仍遵守用户禁止项与仓库规则） | 用户声明「不惜成本」时使用 |
| **auto** | 恢复为 **medium** 等价行为（除非用户或项目规则另有默认） | 同 medium | 撤销显式档位 |

**`max` 会话约束**：仅在用户显式选择 `max` 时启用；用户改选其他档位即降级。

## 宣告格式（必须）

选定档位后，用 **一行** 宣告，便于用户复制到系统提示或后续会话：

`[effort=<level>] 已生效：本会话按 <中文档位名> 执行。`

## 与模型 UI 的关系

若 Cursor / 所用模型提供 **Thinking / Reasoning** 或类似强度控件，应用一句话提醒用户：**在 UI 中选择与 `<level>` 一致的档位**，本命令的文本契约与 UI 同时生效、以较保守者为准（避免 UI 低档 + 文本高档导致越权操作）。

## 禁止

- 不因高档位而绕过 `.agents/rules`、AGENTS.md、或用户对目录/工具的显式禁止。
- 不高档位时自动扩大任务范围（scope creep）。
